# Guides

Guides explain how to apply Tiqlyne APIs to real design and runtime decisions.

- Start with [engine setup](./engine-setup.md) and [registered motions](./registered-motions.md).
- Use [direct timelines](./direct-timelines.md), [timeline positions and labels](./timeline-positions-and-labels.md), [timeline timing options](./timeline-timing-options.md), and [compositions](./compositions.md) for authoring.
- Use [playback controllers](./playback-controllers.md), [events](./events.md), and [reduced motion](./reduced-motion.md) for runtime behavior.
- Use [diagnostics](./diagnostics.md), [sampler](./sampler.md), [inspector](./inspector.md), and [troubleshooting](./troubleshooting.md) when building tools or debugging.
- Extend the engine with [custom definitions](./custom-motion-definition.md) and [custom drivers](./custom-motion-driver.md).
