# Motion timeline

A motion timeline is the low-level animation structure consumed by the engine.

Timelines can be created directly, generated by motion definitions, or compiled from compositions.

## Types overview

| Type                       | Purpose                         |
| -------------------------- | ------------------------------- |
| `MotionTimelineDefinition` | Root timeline object            |
| `MotionTrackDefinition`    | One target's animation track    |
| `MotionStepDefinition`     | One animation segment           |
| `MotionTimelineDefaults`   | Shared timing defaults          |
| `MotionTimelineLabels`     | Named positions in milliseconds |
| `MotionStepPosition`       | Position of a step              |
| `MotionLabelStepPosition`  | Label-based step position       |
| `MotionAnchorStepPosition` | Anchor-based step position      |
| `MotionStepAnchor`         | Anchor type                     |
| `MotionPlaybackDirection`  | Step playback direction         |
| `MotionIterationCount`     | Number of iterations            |
| `MotionFillMode`           | Fill behavior                   |
| `MotionStaggerDefinition`  | Multi-target stagger            |
| `MotionStaggerFrom`        | Stagger origin                  |

---

## `MotionTimelineDefinition`

```ts
type MotionTimelineDefinition = {
  readonly tracks: ReadonlyArray<MotionTrackDefinition>;
  readonly defaults?: MotionTimelineDefaults;
  readonly labels?: Readonly<Record<string, number>>;
};
```

- `tracks` — required. Must contain at least one track.
- `defaults` — optional timing defaults applied to all tracks and steps.
- `labels` — optional named positions in milliseconds.

---

## `MotionTrackDefinition`

```ts
type MotionTrackDefinition = {
  readonly target: MotionTargetReference;
  readonly steps: ReadonlyArray<MotionStepDefinition>;
  readonly stagger?: MotionStaggerDefinition;
  readonly defaults?: MotionTimelineDefaults;
};
```

- `target` — required. Identifies what to animate.
- `steps` — required. Must contain at least one step.
- `stagger` — optional stagger when `target` resolves to multiple elements.
- `defaults` — optional timing defaults for steps in this track.

---

## `MotionStepDefinition`

```ts
type MotionStepDefinition = {
  readonly keyframes: ReadonlyArray<MotionKeyframe>; // required
  readonly duration?: number;
  readonly delay?: number;
  readonly easing?: MotionEasing;
  readonly fill?: MotionFillMode;
  readonly at?: MotionStepPosition;
  readonly iterations?: MotionIterationCount;
  readonly direction?: MotionPlaybackDirection;
  readonly yoyo?: boolean;
  readonly endDelay?: number;
  readonly playbackRate?: number;
  readonly offset?: number;
};
```

- `keyframes` — required. Must contain at least one keyframe.
- `duration` — required either here or in defaults.
- `offset` — low-level field used by drivers and tooling. Must be in range 0–1. Different from keyframe offsets.

---

## `MotionTimelineDefaults`

```ts
type MotionTimelineDefaults = {
  readonly duration?: number;
  readonly delay?: number;
  readonly easing?: MotionEasing;
  readonly fill?: MotionFillMode;
  readonly iterations?: MotionIterationCount;
  readonly direction?: MotionPlaybackDirection;
  readonly yoyo?: boolean;
  readonly endDelay?: number;
  readonly playbackRate?: number;
};
```

Defaults cascade from most specific to least specific: **step → track defaults → timeline defaults**.

---

## `MotionStepPosition`

A step position can take four forms:

```ts
type MotionStepPosition =
  | number // absolute time in ms
  | string // shorthand label name
  | MotionLabelStepPosition // { label, offset? }
  | MotionAnchorStepPosition; // { anchor, offset? }
```

| Form                  | Example                                       | Meaning                            |
| --------------------- | --------------------------------------------- | ---------------------------------- |
| `number`              | `at: 300`                                     | Absolute milliseconds              |
| `string`              | `at: 'intro'`                                 | Shorthand for `{ label: 'intro' }` |
| `{ label, offset? }`  | `at: { label: 'intro', offset: 50 }`          | Label time plus offset             |
| `{ anchor, offset? }` | `at: { anchor: 'previous-end', offset: 100 }` | Anchor-relative position           |

---

## `MotionStepAnchor`

```ts
type MotionStepAnchor = 'track-start' | 'track-end' | 'previous-start' | 'previous-end';
```

| Anchor           | Resolves to                                 |
| ---------------- | ------------------------------------------- |
| `track-start`    | Beginning of the track (0 ms)               |
| `track-end`      | End of the last scheduled step in the track |
| `previous-start` | Start time of the previous step             |
| `previous-end`   | End time of the previous step               |

---

## `MotionPlaybackDirection`

```ts
type MotionPlaybackDirection = 'normal' | 'reverse' | 'alternate' | 'alternate-reverse';
```

| Value               | Behavior                     |
| ------------------- | ---------------------------- |
| `normal`            | Forward playback (default)   |
| `reverse`           | Backward playback            |
| `alternate`         | Alternates forward/backward  |
| `alternate-reverse` | Alternates starting backward |

:::danger Direction ambiguity
`direction` in a step means **playback direction** (`normal`, `reverse`, `alternate`, `alternate-reverse`).
This is different from `slide-in` option `direction` (`left`, `right`, `top`, `bottom`).
Do not confuse them.
:::

---

## `MotionIterationCount`

```ts
type MotionIterationCount = number | 'infinite';
```

Valid values: positive finite number, or the string `'infinite'`. `0`, negative numbers, and `Infinity` are invalid.

---

## `MotionFillMode`

```ts
type MotionFillMode = 'none' | 'forwards' | 'backwards' | 'both' | 'auto';
```

| Value       | Behavior                         |
| ----------- | -------------------------------- |
| `none`      | No fill                          |
| `forwards`  | Hold final value after step ends |
| `backwards` | Apply initial value during delay |
| `both`      | Both forwards and backwards      |
| `auto`      | Driver-defined behavior          |

---

## `MotionStaggerDefinition`

```ts
type MotionStaggerDefinition =
  | number
  | { readonly each: number; readonly from?: MotionStaggerFrom };

type MotionStaggerFrom = 'start' | 'end' | 'center';
```

A number means a fixed delay in milliseconds between each resolved target. The object form also accepts a `from` origin.

## Validation rules

| Rule                                                 | Error code                         |
| ---------------------------------------------------- | ---------------------------------- |
| Timeline must have at least one track                | `timeline-empty-tracks`            |
| Track must have at least one step                    | `timeline-empty-steps`             |
| Step must have at least one keyframe                 | `timeline-empty-keyframes`         |
| `duration` must be finite and ≥ 0                    | `timeline-invalid-duration`        |
| `delay` must be finite and ≥ 0                       | `timeline-invalid-delay`           |
| `endDelay` must be finite and ≥ 0                    | `timeline-invalid-end-delay`       |
| `playbackRate` must be finite and > 0                | `timeline-invalid-playback-rate`   |
| `iterations` must be positive finite or `'infinite'` | `timeline-invalid-iterations`      |
| `direction` must be a valid value                    | `timeline-invalid-direction`       |
| `fill` must be a valid value                         | `timeline-invalid-fill`            |
| `yoyo: true` cannot be combined with `direction`     | `timeline-yoyo-direction-conflict` |
| Step `offset` must be in 0–1                         | `timeline-invalid-step-offset`     |
| Label references must point to existing labels       | `timeline-missing-label-reference` |
| Label times must be finite and non-negative          | `timeline-invalid-label`           |

---

## yoyo and direction conflict

`yoyo: true` is a shortcut for alternate direction behavior. It cannot be combined with an explicit `direction` value on the same step (or in defaults that apply to the same step).

```ts
// ❌ Invalid
track.step({ duration: 400, yoyo: true, direction: 'alternate' }, ...);

// ✅ Use direction when you need precise control
track.step({ duration: 400, direction: 'alternate' }, ...);

// ✅ Use yoyo for simple back-and-forth
track.step({ duration: 400, yoyo: true }, ...);
```

---

## Compatibility rules summary

| Incompatible combination                | Error                              |
| --------------------------------------- | ---------------------------------- |
| `yoyo: true` with any `direction` value | `timeline-yoyo-direction-conflict` |
| `iterations: 0`                         | `timeline-invalid-iterations`      |
| `playbackRate: 0` or negative           | `timeline-invalid-playback-rate`   |
| `endDelay < 0`                          | `timeline-invalid-end-delay`       |

---

## When to use direct timelines

Use direct timelines when you need:

- precise control over multiple tracks;
- custom sequencing, labels, or animation logic;
- animation that is not worth turning into a reusable motion definition.
