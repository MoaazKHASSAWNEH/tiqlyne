# Motion timeline

A motion timeline is the low-level animation structure used by the engine.

Timelines can be created directly, generated by motion definitions or compiled from compositions.

## Structure

A timeline can contain:

- defaults
- labels
- tracks
- steps
- keyframes
- timing options

## Tracks

A track identifies what should be animated.

Common target forms include `self`, child targets, selector targets and named targets.

## Steps

A step defines one animation segment inside a track.

A step can contain timing options and keyframes.

## Labels

Labels identify positions in the timeline.

Playback controllers can use labels when the active driver supports seeking.

## Defaults

Defaults reduce repetition across steps and tracks.

Typical defaults include duration, delay, easing, fill mode, iteration count and playback rate.

## When to use direct timelines

Use direct timelines when you need precise control, multiple tracks, custom sequencing, labels or animation logic that is not worth turning into a reusable motion definition.
