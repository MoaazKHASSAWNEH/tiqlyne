# Diagnostics reference

Diagnostics describe validation, planning or playback issues.

They make engine behavior easier to understand and debug.

## Common fields

A diagnostic can include severity, code, message, source and metadata.

The exact metadata depends on the diagnostic source.

## Sources

Diagnostics can come from validation, planning, scheduling, driver execution or controller operations.

## Use cases

Diagnostics are useful for debugging, tests, builder feedback, custom definition validation and custom driver development.
